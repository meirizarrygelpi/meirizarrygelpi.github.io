Quaternions in Go

Date Sun 03 April 2016 By M.E. Irizarry-Gelpí Category Go Tags quaternions

In order to apply what I have been learning about Go, I decided to write a package for quaternion arithmetic. It is called nion and you can find it on GitHub. My plan is to include traditional quaternions, along with hyperbolic quaternions, split-quaternions, as well as octonions and split-octonions. So far I have some stuff on quaternions and hyperbolic quaternions. In this post I would like to mention some issues I have encountered while working on this project.

A quaternion \(z\) is defined by four real numbers:

$$ z = e_{0} + e_{1} i + e_{2} j + e_{3} k. $$

At first, I thought of implementing a quaternion in Go as an array of float64 values:

type Quater [4]float64

This has the advantage of indexing. Another possibility is to use a struct with four named fields:

type Quater struct {
    E0 float64
    E1 float64
    E2 float64
    E3 float64
}

However, in both of these options the user would be able to set and get the values directly. This sounds nice, but I like the idea of treating the quaternion as a whole, so if you want to change a quaternion, you should create a new one with the desired values. So in the end I went with an encapsulated array:

type Quater struct {
    __ [4]float64
}

This attribute is private, but its name is suggesting of sub-indexing. Getter methods are available:

// E0 method returns the real part of z.
func (z *Quater) E0() float64 { return z.__[0] }

// E1 method returns the imaginary i-part of z.
func (z *Quater) E1() float64 { return z.__[1] }

// E2 method returns the imaginary j-part of z.
func (z *Quater) E2() float64 { return z.__[2] }

// E3 method returns the imaginary k-part of z.
func (z *Quater) E3() float64 { return z.__[3] }

These are convenient to examine individual components of a given quaternion, but I still do not want to include setter methods.

I have included a String method for pretty-printing. This was a royal pain-in-the-butt to write, as I wanted a very particular behavior. The method has almost 180 lines of code and involves 41 return statements:

// String returns the string representation of a Quater.
func (z Quater) String() string {
    if z.__[0] == 0 {
        if z.__[1] == 0 {
            if z.__[2] == 0 {
                if z.__[3] == 0 {
                    return "0"
                }

                return fmt.Sprintf("%gk", z.__[3])
            }

            if z.__[3] == 0 {
                return fmt.Sprintf("%gj", z.__[2])
            }

            if z.__[3] < 0 {
                return fmt.Sprintf("%gj - %gk", z.__[2], -z.__[3])
            }

            return fmt.Sprintf("%gj + %gk", z.__[2], z.__[3])
        }

        if z.__[2] == 0 {
            if z.__[3] == 0 {
                return fmt.Sprintf("%gi", z.__[1])
            }

            if z.__[3] < 0 {
                return fmt.Sprintf("%gi - %gk", z.__[1], -z.__[3])
            }

            return fmt.Sprintf("%gi + %gk", z.__[1], z.__[3])
        }

        if z.__[3] == 0 {
            if z.__[2] < 0 {
                return fmt.Sprintf("%gi - %gj", z.__[1], -z.__[2])
            }

            return fmt.Sprintf("%gi + %gj", z.__[1], z.__[2])
        }

        if z.__[3] < 0 {
            if z.__[2] < 0 {
                return fmt.Sprintf("%gi - %gj - %gk",
                    z.__[1], -z.__[2], -z.__[3])
            }

            return fmt.Sprintf("%gi + %gj - %gk",
                z.__[1], z.__[2], -z.__[3])
        }

        if z.__[2] < 0 {
            return fmt.Sprintf("%gi - %gj + %gk", z.__[1], -z.__[2], z.__[3])
        }

        return fmt.Sprintf("%gi + %gj + %gk", z.__[1], z.__[2], z.__[3])
    }

    if z.__[1] == 0 {
        if z.__[2] == 0 {
            if z.__[3] == 0 {
                return fmt.Sprintf("%g", z.__[0])
            }

            if z.__[3] < 0 {
                return fmt.Sprintf("%g - %gk", z.__[0], -z.__[3])
            }

            return fmt.Sprintf("%g + %gk", z.__[0], z.__[3])
        }

        if z.__[3] == 0 {
            if z.__[2] < 0 {
                return fmt.Sprintf("%g - %gj", z.__[0], -z.__[2])
            }

            return fmt.Sprintf("%g + %gj", z.__[0], z.__[2])
        }

        if z.__[3] < 0 {
            if z.__[2] < 0 {
                return fmt.Sprintf("%g - %gj - %gk",
                    z.__[0], -z.__[2], -z.__[3])
            }

            return fmt.Sprintf("%g + %gj - %gk",
                z.__[0], z.__[2], -z.__[3])
        }

        if z.__[2] < 0 {
            return fmt.Sprintf("%g - %gj + %gk",
                z.__[0], -z.__[2], z.__[3])
        }

        return fmt.Sprintf("%g + %gj + %gk",
            z.__[0], z.__[2], z.__[3])
    }

    if z.__[2] == 0 {
        if z.__[3] == 0 {
            if z.__[1] < 0 {
                return fmt.Sprintf("%g - %gi", z.__[0], -z.__[1])
            }

            return fmt.Sprintf("%g + %gi", z.__[0], z.__[1])
        }

        if z.__[3] < 0 {
            if z.__[1] < 0 {
                return fmt.Sprintf("%g - %gi - %gk",
                    z.__[0], -z.__[1], -z.__[3])
            }

            return fmt.Sprintf("%g + %gi - %gk", z.__[0], z.__[1], -z.__[3])
        }

        if z.__[1] < 0 {
            return fmt.Sprintf("%g - %gi + %gk", z.__[0], -z.__[1], z.__[3])
        }

        return fmt.Sprintf("%g + %gi + %gk", z.__[0], z.__[1], z.__[3])
    }

    if z.__[3] == 0 {
        if z.__[2] < 0 {
            if z.__[1] < 0 {
                return fmt.Sprintf("%g - %gi - %gj",
                    z.__[0], -z.__[1], -z.__[2])
            }

            return fmt.Sprintf("%g + %gi - %gj", z.__[0], z.__[1], -z.__[2])
        }

        if z.__[1] < 0 {
            return fmt.Sprintf("%g - %gi + %gj", z.__[0], -z.__[1], z.__[2])
        }

        return fmt.Sprintf("%g + %gi + %gj", z.__[0], z.__[1], z.__[2])
    }

    if z.__[3] < 0 {
        if z.__[2] < 0 {
            if z.__[1] < 0 {
                return fmt.Sprintf("%g - %gi - %gj - %gk",
                    z.__[0], -z.__[1], -z.__[2], -z.__[3])
            }

            return fmt.Sprintf("%g + %gi - %gj - %gk",
                z.__[0], z.__[1], -z.__[2], -z.__[3])
        }

        if z.__[1] < 0 {
            return fmt.Sprintf("%g - %gi + %gj - %gk",
                z.__[0], -z.__[1], z.__[2], -z.__[3])
        }

        return fmt.Sprintf("%g + %gi + %gj - %gk",
            z.__[0], z.__[1], z.__[2], -z.__[3])
    }

    if z.__[2] < 0 {
        if z.__[1] < 0 {
            return fmt.Sprintf("%g - %gi - %gj + %gk",
                z.__[0], -z.__[1], -z.__[2], z.__[3])
        }

        return fmt.Sprintf("%g + %gi - %gj + %gk",
            z.__[0], z.__[1], -z.__[2], z.__[3])
    }

    if z.__[1] < 0 {
        return fmt.Sprintf("%g - %gi + %gj + %gk",
            z.__[0], -z.__[1], z.__[2], z.__[3])
    }

    return fmt.Sprintf("%g + %gi + %gj + %gk",
        z.__[0], z.__[1], z.__[2], z.__[3])
}

This is probably a classic example of spaghetti code, but it works! It could have been much worst, since there are \(3^{4} = 81\) possible sign combinations. As an example, for the quaternion \(-1 - i - j - k\) it returns -1 - i - j - k. Note the spacing between the binary subtraction operators, and the lack of space between the unary negation operator (look at me throwing jargon around).

Another crazy thing I added was some concurrency by using goroutines and a wait group. Most operations on a quaternion are parallelizable. Here is my version of a factory method that creates new Quater values:

// NewQuater function returns a pointer to a Quater value made from four given
// real numbers (i.e. float64s).
func NewQuater(a, b, c, d float64) *Quater {
    var wg sync.WaitGroup
    z := new(Quater)

    wg.Add(4)
    go func() {
        defer wg.Done()
        z.__[0] = a
    }()
    go func() {
        defer wg.Done()
        z.__[1] = b
    }()
    go func() {
        defer wg.Done()
        z.__[2] = c
    }()
    go func() {
        defer wg.Done()
        z.__[3] = d
    }()
    wg.Wait()

    return z
}

Another example, a method for multiplying a quaternion by a scalar:

// Scalar method sets z equal to s*x, and returns z.
func (z *Quater) Scalar(x *Quater, s float64) *Quater {
    var wg sync.WaitGroup
    a := new(Quater).Clone(x)

    wg.Add(4)
    for i := 0; i < 4; i++ {
        go func(j int) {
            defer wg.Done()
            z.__[j] = s * a.__[j]
        }(i)
    }
    wg.Wait()

    return z
}

The non-concurrent version of this method would be:

// Scalar method sets z equal to s*x, and returns z.
func (z *Quater) Scalar(x *Quater, s float64) *Quater {
    for i, v := range x.__ {
        z.__[i] = s * v
    }

    return z
}

A quick benchmark seems to indicate that my use of the goroutines and wait group leads to slower performance, so I might get rid of it. For the record, the commit with the goroutines and wait groups is 9e39dff7359356b56520863ddf04fee921d0ae51.

Finally, a comment about DRY (Don't Repeat Yourself). As I mentioned, I plan to include quaternions along with hyperbolic quaternions and split-quaternions. All of these are very similar, except for the multiplication method and the fact that the norm of quaternions is real while the norm for hyperbolic quaternions and split-quaternions can be real, imaginary, or vanishing. The obvious thing to do is to copy and paste the quater.go file and promote every Quater type to either a HyperQuater or a SplitQuater type, and change the Norm and Mul methods. That is exactly what I did, but I cannot help but wonder if there is a more natural way to do this and do not repeat myself so much. It is a little tricky, because even though all three quaternion types might have the same methods, I do not want to use an interface since I do not want to be able to for example add two quaternions of different kinds.
